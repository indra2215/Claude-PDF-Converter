## 📥 Installation (Step-by-Step Guide)

### Step 1: Download/Clone the Extension

**Option A: Clone with Git (Recommended)**

1. **Create a folder** where you want to store the extension (e.g., `C:\Extensions` or `~/Documents`)

2. **Open Command Prompt/Terminal** in that folder and run:
   ```bash
   git clone https://github.com/indra2215/Claude-PDF-Converter.git
   ```

3. **Important:** After cloning, you'll see a folder structure like this:
   ```
   Your-Folder/
   ├── .vscode/
   └── Claude-PDF-Converter/  ← This is the extension folder!
   ```

4. **Navigate to the extension folder:**
   ```bash
   cd Claude-PDF-Converter
   ```

**Option B: Download ZIP**

1. Click the green **`<> Code`** button at the top of this page
2. Select **"Download ZIP"**
3. Extract the ZIP file to any folder
4. **Important:** Open the extracted folder and find the `Claude-PDF-Converter` subfolder

---

### Step 2: Install in Chrome/Edge/Brave

<div align="center">
<img src="https://via.placeholder.com/600x400/f3f4f6/1f2937?text=Installation+Visual+Guide" alt="Installation Guide" width="600"/>

#### Detailed Installation Steps:

1. **Open Your Browser Extensions Page**
   - **Chrome:** Click the **3 dots** (⋮) in top-right → **More tools** → **Extensions**
     - OR type in address bar: `chrome://extensions/`
   - **Edge:** Click the **3 dots** (⋯) → **Extensions** → **Manage extensions**
     - OR type: `edge://extensions/`
   - **Brave:** Menu → **Extensions** → **Manage extensions**
     - OR type: `brave://extensions/`

2. **Enable Developer Mode**
   - Look for **"Developer mode"** toggle in the **top-right corner** of the extensions page
   - **Turn it ON** (it will turn blue/green)
   - You'll see new buttons appear: "Load unpacked", "Pack extension", "Update"

3. **Load the Extension**
   - Click the **"Load unpacked"** button (on the left side after enabling Developer mode)
   - A file browser window will open
   - **Navigate to and select the `Claude-PDF-Converter` folder** (the one containing `manifest.json`)
   - Click **"Select Folder"** (Windows) or **"Open"** (Mac/Linux)

4. **Verify Installation** ✅
   - You should see **"ISR-CONVERTER"** card appear in your extensions list
   - It will show:
     - Name: "ISR-CONVERTER - Universal Document Converter"
     - Version: "2.1.0"
     - Status: "Enabled" (blue toggle)
   - You'll see a toast message: **"Extension successfully added!"**

5. **Done!** 🎉
   - The extension is now active and ready to use!
   - No additional configuration needed!

### Step 3: Start Using ISR-CONVERTER!

1. **Open Claude.ai**
   - Go to **[https://claude.ai](https://claude.ai)** in your browser
   - Make sure you're logged in

2. **Upload Any Supported File**
   - You can:
     - **Click** the 📎 attachment icon in Claude's chat input
     - **Drag and drop** a file into the chat window
     - **Paste** an image from clipboard (Ctrl+V / Cmd+V)

3. **Wait for the Magic!** ✨
   - A **toast notification** will appear above the chat input box
   - It will say: **"File detected! Ready to convert"**
   - You'll see the file name and type

4. **Convert to Markdown**
   - Click the **"⚡ Convert to .md"** button in the notification
   - The extension will process your file:
     - **Text files** (PDF, Word, CSV, JSON, XML, HTML): 1-3 seconds ⚡
     - **Images** or **Scanned PDFs**: 5-30 seconds (OCR processing) 🔄

5. **Attach to Claude**
   - After conversion completes, click **"📎 Attach .md to Claude"**
   - The converted Markdown file is automatically attached to your chat
   - You'll see it appear in Claude's message input as an attachment
   - **Now you can chat with Claude about your document!** 💬

6. **See Your Savings!** 💰
   - The notification shows estimated **token savings**
   - Example: "Saved ~10,000 tokens (75%)"

## 🔧 Technical Details

### Architecture

```
File Upload → Type Detection → Smart Routing
                                    ↓
        ┌──────────────────────────┴──────────────────────────┐
        │                                                       │
    PDF Path                  Data Path              Image Path
        │                         │                        │
   pdf.js → Text          JSON/CSV/XML           Tesseract.js
        │     extraction         │                 OCR
        │         ↓              ↓                    ↓
        └─────→ Markdown ←──────┴────────────────────┘
                   ↓
            DataTransfer API
                   ↓
         Attached to Claude! 🎉
```

### Libraries Used

- **pdf.js** - PDF text extraction
- **Tesseract.js** - OCR for images and scanned PDFs
- **mammoth.js** - Word document (.docx) conversion
- Native JS - CSV, JSON, XML, HTML parsing

### Performance

| Format | Speed | Accuracy |
|--------|-------|----------|
| **CSV, JSON, XML, HTML** | ⚡ <1 second | 100% |
| **Word (.docx)** | ⚡ 1-3 seconds | 98% |
| **PDF (text)** | ⚡ 2-5 seconds | 95-100% |
| **PDF (scanned)** | 🐌 5-30 seconds | 70-90% |
| **Images** | 🐌 3-15 seconds | 60-95% |

## ⚠️ Limitations & Tips

### What Works Best

✅ **Text PDFs** - Fast and 99% accurate  
✅ **Word docs** - Clean conversion in seconds  
✅ **CSV files** - Shows ALL data, any size  
✅ **Digital screenshots** - Good OCR accuracy  

### Known Limitations

❌ **Handwritten notes** - OCR not reliable (use Claude's vision instead)  
❌ **Very low-quality scans** - May have errors (extension shows confidence warning)  
❌ **Password-protected PDFs** - Cannot process  
❌ **Old Word format (.doc)** - Only .docx supported (save as .docx first)  

### Smart Recommendations

The extension automatically suggests when to use the original file:
- **Low OCR confidence** (<70%) - Upload original to Claude
- **Very small text** (<50 chars extracted) - Likely failed, use original
- **Handwritten content** - OCR won't work well

### 🐛 Troubleshooting

**Extension Not Working?**

**1. Verify Installation**
```
chrome://extensions/ → Find "ISR-CONVERTER" → Should show version "2.1.0"
```

**2. Wrong Folder Selected?**
If you see an error like "Manifest file is missing" or "Invalid manifest":
- ❌ You selected the wrong folder (probably the parent folder)
- ✅ Make sure you select the `Claude-PDF-Converter` folder that contains:
  - `manifest.json` ← This file MUST be in the folder you select!
  - `content.js`
  - `popup.html`
  - `icons/` folder
  - `lib/` folder

**Folder Structure Explanation:**
```
After cloning/extracting, you might see:
ISR-CONVERTER/                    ← Parent folder (DON'T select this!)
├── .vscode/
└── Claude-PDF-Converter/         ← Select THIS folder! ✅
    ├── manifest.json             ← This file must be here!
    ├── content.js
    ├── popup.html
    ├── icons/
    └── lib/
```

**3. Check Console for Errors**
```
1. Go to claude.ai
2. Press F12 (or Right-click → Inspect)
3. Click "Console" tab
4. Look for: "[ISR-CONVERTER v2.1.0] Extension loaded successfully"
```
- ✅ If you see this message, extension is working!
- ❌ If not, check the steps above

**4. Common Issues & Solutions**

| Problem | Solution |
|---------|----------|
| **Files not detected** | Hard refresh Claude (Ctrl+Shift+R / Cmd+Shift+R) |
| **OCR not working** | Check internet connection (first-time downloads ~4MB model) |
| **Word docs fail** | Make sure it's `.docx` format (not old `.doc`) |
| **Extension disappeared** | Re-enable in `chrome://extensions/` |
| **Notification doesn't appear** | Refresh the Claude.ai page |
| **"Manifest file missing" error** | You selected the wrong folder - select `Claude-PDF-Converter/` |

**5. Still Having Issues?**
- Disable other extensions that might conflict
- Try a different browser (Chrome/Edge/Brave)
- Check if the extension is **Enabled** in `chrome://extensions/`
